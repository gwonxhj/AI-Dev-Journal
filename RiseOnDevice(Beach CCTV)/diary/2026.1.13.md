# 과정
- Odroid m2로 넘어온 후 npu 드라이버 버전 0.9.8에 대부분 고정되어 있는 것으로 확인 -> RKNN toolkit lite2 2.3.2와 완벽하게 호환되지 않음
- 커널 파일만 따로 불러와서 드라이버 업데이트 고려 -> 의존성 문제로 어려움
- toolkit2 2.3.2 강제 사용 시도 -> 호환 문제로 벤치마크 돌릴 시 NPU 드라이버를 찾지 못함 및 toolkit2에서 접근 시도하는 경로의 파일이 존재하지 않는 문제

--> Odroid m2 Ubuntu 20.04, rknn-toolkit-lite2 1.6.0, python 3.8.x, driver version 0.9.8이 현재 최선의 세팅인 것으로 확인 후 세팅

- 워크스테이션 이용 yolov8n, s, m, l_fp32.onnx 파일 rknn 파일로 일괄 순차 변환 중
- Odroid m2 main.cpp 파일 INT8 입출력 형태 고려하여 코드 재구성 후 벤치마크 실시 예정
