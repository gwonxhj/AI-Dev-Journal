# 활동
- 기존 워크스테이션 다른 프로젝트 학습 진행으로 사용 불가능
  -> 새로운 윈도우 PC로 Linux 환경부터 변환 위한 세팅까지 진행

🛠 주요 활동 및 프로젝트 요약 (2026.01.07 ~ 현재)
1. YOLOv8 기반 Edge AI 파이프라인 구축
 - 환경 구성: Ubuntu 20.04 워크스테이션 환경에서 RKNN-Toolkit2 1.6.0 개발 환경 구축 및 가상 환경 최적화.
 - 모델 변환 가속화: PyTorch(.pt) 모델을 NPU에 최적화된 .rknn 포맷으로 변환하는 워크플로우 설계.
   - RKNN 호환성을 고려한 ONNX Opset 12 기반 내보내기 수행.
   - NPU 연산 효율을 높이기 위해 단일 출력 텐서를 좌표(Boxes)와 점수(Scores)로 분리하는 ONNX Graph 수정 스크립트 작성 및 적용.

2. 하이브리드 양자화(Hybrid Quantization) 기술 연구
 - 정확도 손실 해결: 전역 INT8 양자화 시 발생하는 Score Crushing(mAP 0.0000) 현상을 분석하고, 이를 해결하기 위한 정밀도 제어 기법 적용.

 - 혼합 정밀도 최적화:
   - RKNN-Toolkit2의 Custom Hybrid Quantization 모드를 활용하여 백본(Backbone)은 int8로 가속하고, 출력 레이어(Detect Head)는 float16으로 유지하는 최적화 수행.
   - .quantization.cfg 파일의 정책적 제약을 우회하기 위해 custom_quantize_layers 구조를 활용한 동적 패치 스크립트 고도화.

3. Odroid M2(RK3588) 임베디드 벤치마크 개발
 - C++ 추론 엔진 구현: rknn-toolkit-lite2 라이브러리를 활용하여 Odroid M2 보드 전용 C++ 벤치마크 프로그램 개발.
   - RK3588의 Multi-Core NPU를 활용하기 위한 RKNN_NPU_CORE_ALL 마스크 적용.
   - BGR to RGB 전처리 및 Letterbox 알고리즘 최적화 반영.

 - 성능 평가:
   - Inference Time(평균 49ms), FPS(약 20 FPS), Precision, Recall, mAP@50 등 다각도 지표 산출.
   - 실시간 탐지 성능(FPS)과 모델 정확도(mAP) 간의 트레이드오프(Trade-off) 데이터 확보 및 분석.

4. 기술 문서화 및 리포팅
 - 기술 보고서 작성: 비정상적인 좌표값(Overflow) 필터링 로직 및 정규화(Normalization) 파라미터 불일치 해결 과정을 포함한 Technical Report 작성 중.
 - 트러블슈팅 기록: NameError 및 TypeError 등 API 규격 불일치로 인한 런타임 오류 해결 사례 문서화
