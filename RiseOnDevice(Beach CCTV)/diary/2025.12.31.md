# 문제
- 기존 Ubuntu 20.04 사용하다가 22.04로 업데이트 -> NPU 드라이버 포함 안 된 버전으로 받음
  -> 기존 FPS 13내외 -> 4 내외로 추락, 추론이 NPU로 돌아가지 않아서 그런것으로 추정
- 커널 버전 6.1.0 사용중이었는데 해당 커널은 NPU 드라이버 파일 구하는데 어려움 있음
  -> 커널 버전 5.10으로 다운그레이드 후 NPU 드라이버 파일까지 올렸는데 Odroid에서 NPU를 잡지 못함
- Cma 용량 부족을 원인으로 생각하고 config.ini 파일에 cma=256M 삽입하여 부트 시도했는데 무한부팅 현상 발생
  -> extlinux.conf 파일에 아래 문구 삽입하여 cma 용량 확보 성공
  --> LABEL OdroidM1-5.10-NPU
      KERNEL /vmlinuz-5.10.0-odroid-arm64
      INITRD /initrd.img-5.10.0-odroid-arm64
      FDT /dtbs/5.10.0-odroid-arm64/rockchip/rk3568-odroid-m1.dtb
      APPEND root=UUID=여기에_아까_복사한_UUID_입력 rw rootwait console=ttyS2,1500000 console=tty0 cma=256M
- 용량 확보는 성공하였는데 npu가 여전히 안잡혀서 FDT 라인 odroid-m1-npu.dtb 수정했는데 마찬가지 무한 부팅 문제

이후 관련하여 프로젝트 진행해볼것
