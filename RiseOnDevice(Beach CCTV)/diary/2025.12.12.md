## Odroid M1에서 INT8로 양자화한 RKNN 파일을 이용하여 Cpp 환경에서 추론하기 위한 작업 시작
- 최근 C++로 추론했던 RKNN 파일은 FP16 파일이었음
- Ultralytics를 이용하여 작업 시작

## 과정
- Ubuntu 20.04 버전 설치 된 워크스테이션에서 Ultralytics를 이용하여 yolov8n_INT8.rknn 파일 만듦
- FPS 값만 출력되고 나머지 항목 점수 안 나오는 문제 발생

## 정보 기반 문제 해결 시도
- FP16은 색상 채널이 뒤바뀌어도(BGR로 돌아가도) 모델이 형상(Shape)을 인식하여 어느 정도 검출해내는 경우가 있다고 함
- INT8 양자화는 각 채널(R, G, B)의 픽셀 값 분포(Histograms)를 분석해서 압축함
 -> R 채널의 통계정보로 압축된 모델에 B 채널 데이터를 집어넣으면, 값의 분포가 완전히 틀어져서 정확도가 0%로 급락함
- 위 정보 기반 main.cpp의 이미지 로드 부분에 아래 코드 넣고 실행했으나 동일 문제 발생
 -> cv::cvtColor(img, img, cv::COLOR_BGR2RGB);
- 디버깅용 코드 추가하여 추론 과정에서 이미지 5장 단위로 학습 점수 출력해보았으나 '0'으로 출력됨
- 처음 캘리브레이션 데이터셋에 val 이미지 20장만 넣어서 했다가 Jetson 조에서 val 폴더 이미지 전체를 넣어 진행했다고 하여 1600장으로 변경
 -> 캘리브레이션 데이터셋 변경 후 pt -> onnx -> rknn 파일로 다시 변환하여 진행했으나 동일 문제 반복
